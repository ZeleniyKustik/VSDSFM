В этом файле описывается то как хранятся различные данные для работы VSDSFM.
Это написано в блоках хранилищах после порта

ВНИМАНИЕ!!! На каждом блоке хранилища (кроме порта) в 0 ячейке написан id блока по которому процессоры должны подключатся к нему.
Например ядру нужно что-то прочитать из раздела находящимся в блоке хранилище под id 42.
После чего ядро прочитало id с 0 ячейки. Там написано 21. Следовательно при подключении процессоров и блоков хранилища произошла ошибка.



0   | [75124503237205] Это число означает что система настроена
1   | [91557198384710] Это число говорит об том что дальше идёт информация об системе
2   | [0] Здесь написан номер версии
3   | [ ] Количество ячеек в 1 хранилище
4   | [ ] Количество блоков хранилищ
5   | [ ] Количество блоков хранилищ для порта
6   | [ ] Количество ядер для чтения/записи
7   | [ ] Максимальное количество разделов
8   | [ ] Количество занятых разделов
9   | [ ] Общая ёмкость хранилища
10  | [ ] Сколько ячеек занято
11  | [ ] размер системного кэша (нужен чтобы система не использовала память порта)
12  | [a] С какой ячейки начинаются мета данные об всех разделах
13  | [b] С какой ячейки начинается массив общения с ядрами
14  | [c] С какой ячейки начинается массив взаимодействия ядер
15  | [d] С какой ячейки начинается основная память


Хранение метаданных об разделах
a   | [ ] id раздела
a+1 | [ ] Ячейка с которй начинается раздел
a+2 | [ ] Последняя ячейка раздела

a+3 | [ ] id раздела
a+4 | [ ] Ячейка с которй начинается раздел
a+5 | [ ] Последняя ячейка раздела
...


место для общения с ядрами
b   | [1] (1 можно вписать ТОЛЬКО после того как все данные были написаны)
b+1 | [ ] Номер команды
...

Место где хранится память
c   | [ ] Просто число
c+1 | [ ] Просто число
c+2 | [ ] Просто число
c+3 | [ ] Просто число
...
